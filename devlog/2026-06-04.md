# Dev log — 2026-06-04

## What now works (end-to-end)

- **Camera** streams MJPEG over Wi-Fi (`firmware/unitcams3_stream`). Two physical cams,
  one firmware, distinguished only by `HOSTNAME` (`sidekick-loop` = eye for live, `sidekick-cam` = data collector).
- **Dataset collection**: `firmware/pantilt_sweep` auto-sweeps the gimbal; `collector/sidekick_collector.py`
  reads the stream, keeps only **still** frames (frame-diff), saves numbered JPEGs, serves a viewer.
- **Live brain loop**: `brain/live_loop.py` drives the gimbal over serial and scores each view with the
  **9-dimension composer** (`brain/live_brain.py` → Claude scores taste dims; `worth = compose(dims,weights)`),
  optionally blended 50/50 with a **content match** ("animals", "photos"). Search when worth is low,
  **stop + dwell** when high, get bored after a cooldown then **turn away** (with a refractory so it
  doesn't re-lock the same thing). Web control panel at `localhost:8090`: live dim bars, weight tuning,
  threshold/cooldown sliders, event sounds.
- **Voice**: `brain/voice_agent.py` — laptop mic, wake word **"hey potato"** (VAD captures whole
  utterances), then say a noun ("animals" → content) or "more story / less clutter" (→ dimension weights).
- **Pan-tilt** `firmware/pantilt_r4` is pure muscle (laptop-driven) and **auto-relaxes** servos when
  idle (no buzz/heat).

## Key decisions

- **Noticing-first, not object-finding** — the thesis. Judgment = 9 theory dimensions (Berlyne/Kaplan/
  craft) weighted; preference is an optional lean. Content/object match added on top (VLM zero-shot) so
  specific things can still be noticed.
- **Voice = laptop mic, not the device PDM mic.** Continuous wake-word listening belongs on the laptop;
  streaming device audio would fight the video. (Device-mic experiment dropped — files removed.)
- Composer is **linear** (`worth = Σ w·dim`) on purpose → lab dataset can regress the weights + threshold later.

## Gotchas learned

- M5 CamS3-5MP (PY260) only streams at **UXGA** with the official web_cam firmware; hand-rolled
  `esp_camera_init` (SVGA/5MP) init-OK but never delivered frames. Board must be **M5UnitCAMS3 + OPI PSRAM**.
- Camera serves **one viewer at a time** — don't open its IP in a browser while the loop/collector reads it.
- Pan-tilt: servos need an external **5V 2–3A** supply (USB bus browns out); the **R4 must be on USB**
  (serial) for the loop, not just a power bank.
- Voice/Whisper: drop the first ~1s mic warm-up; normalize; transcribe the file (not a raw array);
  a 32-bit DC-sum overflowed past ~4s of audio (now 64-bit).
