# gaze.py 测试清单 — 四个注意力关系

顺序:webcam(30fps,先排逻辑问题)→ M5(2fps,只验"低帧率下还能不能用")。
每项都给:怎么做 / 通过标准 / 不过调哪个旋钮。

```bash
python gaze.py --camera 0                  # 0–2 节:只看射线,不开检测
python gaze.py --camera 0 --detect         # 3 节起:加检测器
```

## 0. Smoke(1 分钟)
- [ ] 窗口打开,显示 fps;一张脸 → 1 条黄射线 + yaw/pitch 数字
- [ ] fps:webcam 上 FaceMesh+Pose 同开 ≥ 15fps(不够就 `--no-arms` 看是不是 Pose 拖的)

## 1. 射线质量(gaze ray 本身)
- [ ] **方向跟随**:头转左/右/上/下,射线同向;yaw 左转为负右转为正(或反——记下符号约定)
- [ ] **稳定性**:头不动,射线抖动 < ~5°(肉眼箭头端点不乱跳)。抖得厉害 → 之后给 gate 加帧间平滑,先记录
- [ ] **正对相机**:射线变短/方向乱是预期的(2D 退化)——确认此时 EYE CONTACT 接管,不算 bug
- [ ] **距离极限**:人逐渐后退,记下 FaceMesh 丢脸的距离(webcam 和 M5 各记一次)。**这是 M5 房间尺度可行性的关键数字**
- [ ] **多人**:两张脸 → 两条射线,各自跟随

## 2. eye-contact
- [ ] 画面中央看镜头 → 白框 "EYE CONTACT",移开视线 → 消失
- [ ] **偏离中心**:站到画面边缘看镜头 → 仍触发;看正前方(不看镜头)→ 不触发(这是 3D 测试对 2D 的优势,务必验)
- [ ] **闪烁率**:保持注视 10 秒,数掉线次数。≥2 次/10s → 调 `eye_contact(tol_deg=...)`(12→15)
- [ ] 角度数字随头转平滑增大,~12° 处切断

## 3. #5 gazing-at(--detect)
- [ ] 桌上摆 2–3 个物体(cup/bottle/book),看其中一个 → 绿框选对;换看另一个 → 切换
- [ ] **主体绑定**:绿框出现时,青色框框住你(person);YOLO 没检出人时降级为 face 框,不崩
- [ ] **不该触发**:看两物体之间的空处 → 无绿框。误触 → `--tol` 12→8
- [ ] **该触发**:看物体边缘 → 仍触发。漏触 → `--tol` 12→15
- [ ] **切换迟滞**:视线在两物体间来回,绿框跟得上吗?会不会在中间乱跳?(乱跳频率 = 之后 gate 习惯化的压力测试输入,记录)
- [ ] 检测器 A/B:`--detector yolo` vs `gdino` — 同一场景比较目标框稳定性和命中正确率(gdino 卡是预期)

## 4. #4 pointing-at
- [ ] **触发过滤**:手臂自然下垂/弯曲 → 无橙色射线;伸直指向 → 出现。这是本关系的核心,反复验
- [ ] 阈值手感:半伸(~150°)该不该算?不顺手 → `ArmRayEstimator(min_extension_deg=...)` 140±10
- [ ] 指向桌上物体 → 绿框 "pointing-at" 选对;指向另一人 → 触发(#4 允许人作目标)
- [ ] 左右臂各测;侧身(手臂前后向)时 2D 射线还可信吗?记录失效姿态
- [ ] **多人**:两人同时入画各自伸臂 → 两套臂射线都出现(tasks API PoseLandmarker,num_poses=4)

## 5. #6 joint-attention(需两人)
- [ ] 两人看同一物体 → 紫圈出现在物体附近
- [ ] 两人对视 → 不该出 joint-attention(交点在脸上,退化拒绝)…但这其实是 eye-contact 的人-人版,观察实际行为并记录
- [ ] 两人看不同方向 → 无紫圈
- [ ] 紫圈位置稳定性:共同注视 10 秒,圈心漂移多大(给 gate 的"同一目标"判定定空间容差)

## 6. M5 复测(--camera http://<ip>/)
- [ ] 0/1/2 节重跑:2fps 下 FaceMesh video-mode 追踪还有效吗(帧间隔太大可能退化,若丢 → `HeadPoseEstimator` 换 static_image_mode=True 试)
- [ ] 房间尺度距离极限(1 节);实验室顶光/逆光下的脸检率
- [ ] 只开一个 viewer(M5 单连接)

## 产出(给下一步 gate 设计用的数字)
1. 各关系的**闪烁率**(帧间翻转频率)→ 决定 TemporalConfigGate 的 persist/window 参数
2. FaceMesh **距离极限**(M5)→ 决定扫描时机器人离人多远有效
3. 调定的阈值:gazing tol / eye-contact tol / min_extension_deg / pointing tol
4. 失效姿态清单(侧身指、低头看手机时的 gaze…)→ 写进论文 limitations
